languages/frameworks/source control

These are the languages, frameworks and source control that I have worked with in my career:

  • React

    React is a free and open-source front-end JavaScript library that aims to make building user interfaces based on components more "seamless".

  • PHP

    PHP is a general-purpose scripting language geared toward web development.

  • Laravel

    Laravel is a free and open-source PHP web framework, intended for the development of web applications following the model-view-controller architectural pattern and based on Symfony.

  • Python

    Python is an interpreted high-level general-purpose programming language. Python's design philosophy emphasizes code readability with its notable use of significant indentation.

  • Django

    Django is a Python-based free and open-source web framework that follows the model–template–views architectural pattern.

  • C#

    C# is a general-purpose, high-level multi-paradigm programming language. C# encompasses static typing, strong typing, lexically scoped, imperative, declarative, functional, generic, object-oriented, and component-oriented programming disciplines.

  • .NET

    The .NET Framework is a proprietary software framework developed by Microsoft that runs primarily on Microsoft Windows. It was the predominant implementation of the Common Language Infrastructure until being superseded by the cross-platform .NET project.

  • Ruby on rails

    Ruby on Rails, or Rails, is a server-side web application framework written in Ruby under the MIT License. Rails is a model–view–controller framework, providing default structures for a database, a web service, and web pages

  • Git

    Git is software for tracking changes in any set of files, usually used for coordinating work among programmers collaboratively developing source code during software development. Its goals include speed, data integrity, and support for distributed, non-linear workflows.

  • Docker

    Docker is a set of platform as a service products that use OS-level virtualization to deliver software in packages called containers. Containers are isolated from one another and bundle their own software, libraries and configuration files; they can communicate with each other through well-defined channels.
